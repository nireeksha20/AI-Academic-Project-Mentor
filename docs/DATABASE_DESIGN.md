# 🗄️ Database Design

# AI Academic Project Mentor

**Version:** 1.0

---

# 1. Overview

The application uses **MongoDB**, a NoSQL document database, to store user information, project details, AI-generated outputs, and mentoring conversations.

The database is designed to be modular, scalable, and aligned with the Multi-Agent Architecture.

---

# 2. Database Collections

The system consists of five primary collections.

- Users
- Profiles
- Skill Assessments
- Projects
- Chats

---

# 3. Collection Design

## 3.1 Users

Stores authentication information.

Fields

- \_id
- name
- email
- password
- createdAt
- updatedAt

---

## 3.2 Profiles

Stores academic information.

Fields

- profileId
- userId
- college
- department
- year
- skills
- interests

---

## 3.3 Skill Assessments

Stores assessment responses.

Fields

- assessmentId
- userId
- programmingSkills
- frameworks
- aiKnowledge
- confidenceLevel
- completedAt

---

## 3.4 Projects

Stores everything related to a student's project.

Fields

- projectId
- userId
- projectTitle
- projectIdea
- requirements
- blueprint
- technologyRecommendation
- databaseDesign
- roadmap
- learningPlan
- status
- createdAt
- updatedAt

---

## 3.5 Chats

Stores mentoring conversations.

Fields

- chatId
- projectId
- userId
- sender
- message
- timestamp

---

# 5. Data Ownership

Each collection has a clear responsibility.

Users

Authentication

Profiles

Academic Information

Skill Assessments

Student Skills

Projects

Project Details and AI Outputs

Chats

Conversation History

---

# 6. AI Generated Data

The following outputs generated by the Multi-Agent Pipeline are stored inside the Project document.

- Blueprint
- Technology Recommendation
- Database Design
- Roadmap
- Learning Plan

This design keeps all project-specific information together while reducing unnecessary collections.
